Json序列化与反序列化框架-Gson 入门

Posted on 2018-11-27，5 min read

前言

Gson 是 Google 开源的用于对象针对 JOSN序列化与反序列化的 Java 框架.

优势在于

  • 性能高
  • 标准化
  • 支持泛型
  • 使用简单

本文将对 Gson 框架中的 API 使用做些简单的介绍.

正文

使用步骤

  1. 利用 GsonBuilder 创建 Gson 对象
  2. 使用 Gson 对象的 fromJson 将JSON 数据转变成对象.
  3. 利用 Gson 对象的 toJson 将对象转变为 JSON 字符串

常用 API

  • toJson
  • fromJson

对象序列化

对象序列化成 JSON 文件到本地

GsonBuilder builder = new GsonBuilder(); 
Gson gson = builder.create(); 
FileWriter writer = new FileWriter("student.json");   
writer.write(gson.toJson(student));   
writer.close();

从本地文件反序列化成对象

GsonBuilder builder = new GsonBuilder(); 
Gson gson = builder.create(); 
BufferedReader bufferedReader = new BufferedReader(new FileReader("student.json"));   
Student student = gson.fromJson(bufferedReader, Student.class);

数据绑定

原生数据绑定: Java 内置类: String,List,Map等

name = gson.fromJson("\"Mahesh Kumar\"", String.class);
rollNo = gson.fromJson("1", Long.class);
verified = gson.fromJson("false", Boolean.class);
marks = gson.fromJson("[100,90,85]", int[].class);

对象数据绑定: 任何Java 对象.

String jsonString = gson.toJson(student);          
Student student1 = gson.fromJson(jsonString, Student.class);

TreeModel

在内存中生成JSON 文档的树形结构

解析成 TreeModel

JsonParser parser = new JsonParser();
JsonElement rootNode = parser.parse(jsonString);

遍历 TreeModel

JsonObject details = rootNode.getAsJsonObject();
JsonElement nameNode = details.get("name");
System.out.println("name:" + nameNode.getAsString());

Streaming API

将 JSON 内容的读写视为离散事件, JsonReader 和 JsonWriter 读写的数据视为 Token,成为 JsonToken.处理JSON方法中最强大的方法。它具有最低的开销，并且在读/写操作中非常快

JsonReader reader = new JsonReader(new StringReader(jsonString));
//start reading json   
reader.beginObject(); 

//get the next token 
JsonToken token = reader.peek(); 

//check the type of the token 
if (token.equals(JsonToken.NAME)) {     
   //get the current token 
   fieldname = reader.nextName(); 
}

reader.endObject();

泛型处理

通常获取要映射Json文本的对象类型时,泛型对象的信息丢失,为此 Gson 提供com.google.gson.reflect.TypeToken类来存储泛型对象的类型。

Type type = new TypeToken<T>() {}.getType();
//pass the type of collection
marks = gson.fromJson(jsonString, type);

使用自定义 Adapters

  1. 实现 TypeAdapter的方法 read/write
  2. GsonBuilder 实例注册自定义 TypeAdapter
// 将 Java 对象转换成 Json 串
public abstract void write(JsonWriter out, T value) throws IOException;

// 将 json 串转换成 Java 对象
public abstract T read(JsonReader in) throws IOException;

Null 对象支持

Gson默认忽略 null 的值,可以通过**GsonBuilder.serializeNulls()**来显示 null 值.

GsonBuilder builder = new GsonBuilder(); 
builder.serializeNulls();

版本支持

提供@ Sine 注解控制 类的JSON 序列化与反序列化的版本

class Student { 
   @Since(1.0) 
   private int rollNo; 
   
   @Since(1.0) 
   private String name; 
   
   @Since(1.1) 
   private boolean verified;  
}

同样用 GsonBuilder 方法 setVersion 来处理指定版本的字段.

GsonBuilder builder = new GsonBuilder(); 
builder.setVersion(1.0);   
Gson gson = builder.create();

指定字段的序列化排除

默认 Gson 排除 transient 和 static 修饰的字段进行序列化;但提供了 GsonBuilder#excludeFieldsWithModifiers来控制需要排除特定修饰符修饰的字段.

GsonBuilder builder = new GsonBuilder(); 
builder.excludeFieldsWithModifiers(Modifier.TRANSIENT);    
Gson gson = builder.create();

并且 Gson 提供了@ Expose 注解来控制一个类的序列化和反序列化的字段, 配合**GsonBuilder.excludeFieldsWithoutExposeAnnotation()**使用,表示只暴露特定的变量进行序列化与反序列化.

JSON 字段自定义

使用注解@SerializedName

private class SomeObject {
  @SerializedName("custom_naming") private final String someField;
  private final String someOtherField;

  public SomeObject(String a, String b) {
    this.someField = a;
    this.someOtherField = b;
  }
}

SomeObject someObject = new SomeObject("first", "second");
Gson gson = new GsonBuilder().setFieldNamingPolicy(FieldNamingPolicy.UPPER_CAMEL_CASE).create();
String jsonRepresentation = gson.toJson(someObject);
System.out.println(jsonRepresentation);
//{"custom_naming":"first","SomeOtherField":"second"}
