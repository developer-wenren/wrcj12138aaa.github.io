SpringBoot实战-初体验

2018-10-12 / 7 min read

前言

在分布式时代潮流中, 随着微服务的流行, Spring Boot 作为用来快速交付微服务的Java开源框架名声大噪,Spring Boot依托于请强大的Spring Framework,其优势在于

  • 快捷开发具备生产能力的Spring程序

  • 提高生产力,减少开发效率

  • 自动化配置,没有繁琐的XML配置

这篇文章将以实战方式创建一个基于Spring Boot搭建的简单Rest服务程序,来学习下如何开发Spring Boot程序.

正文

Spring Initializer

Spring Initializer 是官方提供用来快速启动Spring Boot程序的工具,在Spring Initializer网站上,我们可以选择Spring Boot的版本和平台,并且需要提供一系列Maven坐标GroupId,ArtifactId 和程序所需要的依赖.

如下图所示,我们现在创建一个SpringBoot的Maven工程,使用Java定义语言, 版本为2.0.5,定义好Group和Artifact,在 Dependencies 依赖项一栏搜索Web,添加 spring-boot-starter-web依赖来搭建一个Web项目.

SpringBoot 2.0.5 依赖

Java 8

Spring Framework 5.0.9

点击生成工程, Spring Initializer 就帮我们生成一个有Web环境依赖的项目压缩包,解压后就可以直接用IDEA打开使用,也就是所说的开箱即用.

Maven配置

首先我们打开项目里pom.xml文件,来看下项目Maven配置有什么特别之处.

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
	<modelVersion>4.0.0</modelVersion>

	<groupId>com.one</groupId>
	<artifactId>springboot-quickstart</artifactId>
	<version>0.0.1-SNAPSHOT</version>
	<packaging>jar</packaging>

	<name>springboot-quickstart</name>
	<description>Demo project for Spring Boot</description>
    
	<parent>
		<groupId>org.springframework.boot</groupId>
		<artifactId>spring-boot-starter-parent</artifactId>
		<version>2.0.5.RELEASE</version>
		<relativePath/> <!-- lookup parent from repository -->
	</parent>

	<properties>
		<project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
		<project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>
		<java.version>1.8</java.version>
	</properties>

	<dependencies>
		<dependency>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-starter-web</artifactId>
		</dependency>

		<dependency>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-starter-test</artifactId>
			<scope>test</scope>
		</dependency>
	</dependencies>

	<build>
		<plugins>
			<plugin>
				<groupId>org.springframework.boot</groupId>
				<artifactId>spring-boot-maven-plugin</artifactId>
			</plugin>
		</plugins>
	</build>
</project>

Spring Initializer生成的项目自动给我们添加测试环境的依赖:

<dependency>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-starter-test</artifactId>
			<scope>test</scope>
</dependency>

而Web环境的依赖,是我们生成前手动关联的,所有也有对应配置:

<dependency>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-starter-web</artifactId>
</dependency>

还有值得注意一点的是, Maven配置有parent节点坐标,这说明该项目是以父pom文件的配置为基础上进行的定制.

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.0.5.RELEASE</version>
    <relativePath/> <!-- lookup parent from repository -->
</parent>

Main方法

这里我用IDEA打开生成的项目,发现只有一个Java类 SpringbootQuickstartApplication,只有一个Main方法, 而这个类正是是SpringBoot程序的启动类.

/**
 * @author One
 */
@SpringBootApplication
public class SpringbootQuickstartApplication {

	public static void main(String[] args) {
        SpringApplication.run(SpringbootQuickstartApplication.class, args);
	}
}

SpringbootQuickstartApplication类用 @SpringBootApplication注解修饰,来表明他是个SpringBoot程序.

实现Rest请求

现在我们可以来实现能处理HelloWorld的Rest请求的SpringBoot程序,为了简单起见,直接在 SpringbootQuickstartApplication类上进行修改即可.

  1. 类上添加 @RestController注解,表明该类是一个处理Rest请求的Controller
  2. 实现一个请求处理方法 hello() 使用注解 @RequestMapping表明所要处理对应的请求路径
  3. 请求处理方法 hello()直接返回 Hello World 字符串,作为Rest请求的响应
/**
 * @author One
 */
@SpringBootApplication
@RestController
public class SpringbootQuickstartApplication {

	public static void main(String[] args) {
		SpringApplication.run(SpringbootQuickstartApplication.class, args);
	}

	@RequestMapping(value = "/")
	public String hello() {
	    return "Hello World";
	}
}

启动SpringBoot

简单的编码完成后,就可以运行程序了. 如果在IDEA环境下,这个程序可以直接通过运行Main方法来启动,这样比较简单就多说.还有一种情况就是以jar 包的形式启动,这里具体介绍下这种启动方式.

  1. 首先我们可以利用 Maven命令生成程序的可执行jar,在项目源路径下输入下命令:
mvn clean install

执行命令后,看到BUILD SUCCESS信息后,就可以在target目录下看到一个可执行jar文件: springboot-quickstart-0.0.1-SNAPSHOT.jar

  1. 使用java命令运行jar包
java -jar springboot-quickstart-0.0.1-SNAPSHOT.jar

当控制台出现下图内容时就表示程序启动成功了.

从控制台信息可以看到,Spring Boot程序已经部署在Tomcat上了,且服务端口为8080(http),现在就可以打开浏览器,输入 http://localhost:8080/ 访问.

到此,一个最简单的处理Rest请求的Spring Boot程序就完成了. 🎉🎉🎉
